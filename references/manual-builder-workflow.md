# 工业设备说明书技能制作流程

适用范围：工业设备、非标设备、烫印/热转印设备、输送与提升设备、中英双语 HTML 说明书，以及需要 SFC 时序确认和 HMI 参数解释的客户交付文件。

核心目标：把一次说明书制作流程沉淀成可复用、可审计、可开源的技能。最终客户只交付 HTML；截图和双语标注图分别是内部验证证据与 HTML 资源，不作为额外客户交付格式。本文件只作为技能内部 workflow reference。

## 0. 启动原则

1. 先建项目管理，再写正文。
2. 先理清动作时序，再解释 HMI 参数。
3. 先保留真实图片，再补英文注释。
4. 先做客户可读文本，再把内部联动逻辑放进项目备忘。
5. 不编造未确认的温度、压力、速度、产能、PLC 内部逻辑或隐藏模式。

## 1. 项目文件夹

建立或整理以下目录：

| 目录 | 用途 |
|---|---|
| `设备图片` | 整机图、结构标注图 |
| `物理调节` | 气源、压力、胶轮、膜路、收卷、高度等机械调节图 |
| `HMI界面` | HMI 运行界面、参数页、模式页 |
| `双语标注图` | 原图基础上的中英双语标注图 |
| `源稿` | 图片注释清单、章节结构、A4 分页规划、参数表、素材清单 |
| `项目管理` | 项目看板、交接摘要、SFC 审核稿、反馈处理记录、知识沉淀 |
| `输出` | 浏览器截图、静态审计和验证记录 |
| `临时文件` | 图片处理脚本、临时截图、试验文件 |

## 2. 项目看板

`项目管理/项目看板.md` 必须记录：

- 项目路径和最终交付类型。
- 文件夹结构与用途。
- 交付原则。
- 当前任务状态。
- 已识别资料。
- 收尾记录和验证输出。

看板不是装饰文件，它用于防止素材、版本、反馈和输出物混在一起。

## 3. 素材盘点

按证据类型整理源文件：

| 类型 | 典型文件 | 后续用途 |
|---|---|---|
| 整机结构 | 整机图、结构标注图 | 设备概览、机构名称 |
| 机械调节 | 通气、压力、受力平衡、胶轮、膜路、收卷 | 快速上手、机械调节章节 |
| HMI 参数 | 运行页、参数页 | HMI 参数卡、SFC 映射 |
| 手动/IO | 手动按钮、IO 监控、电路/气路 | 手动调试、安全与故障排查 |
| 反馈 | 用户批注截图或文档 | 修订方向、最终口径 |

输出前要保证每个进入 HTML 的图片都能在项目内用相对路径访问。

## 4. 图片双语注释

处理顺序：

1. 判断图片是否已有英文 UI 或英文标签。
2. 对只有中文注释的图片，先写 `源稿/图片双语注释清单.md`。
3. 使用生图模型时，只用来辅助英文表达或布局判断；若模型重绘设备导致结构失真，正式图必须回到原图本地叠字。
4. 输出到 `双语标注图`，文件名保持原图语义，例如 `通气_双语.png`。
5. 在 HTML 中引用双语图，不引用临时模型样稿。

英文注释应短、具体、贴近动作，例如：

- Insert air hose.
- Pull up and rotate to adjust air pressure.
- Adjust the photoelectric switch position.
- Use low speed first and confirm no interference.

## 5. SFC 时序审核

这是最关键关卡。没有 SFC，就不要把延时、速度、收卷、回位、光电等参数写成确定的客户解释。

SFC 审核稿必须包含：

1. 审核目标。
2. 证据来源。
3. 当前理解的动作总览。
4. 主时序表。
5. HMI 参数到 SFC 的映射。
6. 手动动作到 SFC 的映射。
7. 关键审核问题或审核后确认点。
8. 审核后的写入方式。

时间参数必须写明时间原点。默认优先把“按下启动按钮 / Start”作为 T0；如果用户或 PLC 证据确认另有触发点，以确认稿为准。

## 6. HMI 参数解释

本技能已内置 HMI/SFC 参数解释流程，完整细则见 `references/hmi-parameter-explanation.md`。制作说明书时不需要再调用单独的 HMI 参数解释技能。

每个参数按以下顺序写：

1. 中文名 + 原 UI English。
2. 所属 SFC 步骤。
3. 控制的动作或机构。
4. 调大/调小或打开/关闭的结果。
5. 什么时候调整。
6. 调整后的验证方法。
7. 必要安全提醒。

客户 HTML 中不写“参数联动”“PLC 备忘”“内部推断”。这些内容可以进入 `项目管理/参数联动关系_PLC备忘.md`，供后续 PLC/SFC 梳理复用。

## 7. HTML 章节

推荐章节：

1. 设备概览 / Machine Overview
2. 安全事项 / Safety
3. 快速上手 / Quick Start
4. 机械调节 / Mechanical Adjustment
5. HMI 参数 / HMI Parameters
6. SFC 自动循环时序 / SFC Automatic Sequence
7. 手动调试 / Manual Debugging
8. 故障排查 / Troubleshooting
9. 附录 / Appendix

设计方向：

- 工业说明书，不做营销式落地页。
- 所有设备使用同一套 A4 单页容器，不依赖自动分页，也不另建手机页面框架。
- A4 页面按视口等比例适配，页面数量、结构和内容顺序不因设备改变。
- 详细机构图和 HMI 在 A4 内容区内至少占宽度约 75%；优先通栏或独立页，不挤入两个窄栏。
- 图片大，参数卡短，SFC 表清楚。
- 中英双语成对出现。
- UI 原英文保留，后接更清楚的英文解释。

## 8. 反馈处理

收到反馈后按类型处理：

| 反馈类型 | 先改哪里 | 再改哪里 |
|---|---|---|
| SFC/时序 | `项目管理/SFC时序审核稿.md` | HTML 参数、SFC、故障排查 |
| 参数口径 | 参数映射或备忘 | HTML 参数卡 |
| 图片标注 | 图片注释清单 | 双语标注图、HTML 引用 |
| 视觉排版 | HTML/CSS | 截图验证 |

重要反馈要保留备份版本，但最终交付文件名要清楚，不让客户打开草稿或旧版。

## 9. 输出验证

最终至少验证：

- HTML 能直接打开。
- 图片引用全部存在。
- 图片路径为相对路径。
- 中文解释与英文解释成对出现。
- 无 `草稿`、`待审核`、`TODO`、`内部`、`源稿` 等客户不可见字样。
- SFC 关键时序已吸收反馈。
- 手机首屏和长页可读。
- A4 页面无内容溢出；手机和 PC 页面结构一致；主图在 A4 中占比足够大。
- 最终客户文件只有一个已批准 HTML；验证截图保留在 `输出` 内部目录。

## 10. 可复用知识沉淀

沉淀时分层：

- 调机机理、故障判断、时间原点、收卷选择：进入或同步到 `hotfoil-tune`。
- HMI 参数、SFC 映射、客户解释模板：进入 `mg-industrial-equipment-manual-builder` 的内置 HMI 参数解释参考。
- 说明书项目结构、章节模板、确认关卡、验证方法：进入 `mg-industrial-equipment-manual-builder`。
- 经确认的设备专用术语也进入 `mg-industrial-equipment-manual-builder` 的参考材料，避免依赖外部旧技能。

## 11. 技能内阶段关卡

这些阶段用于驱动技能执行。若用户要求一步步确认，按以下阶段提交；若用户要求全程执行，则把每一阶段写入项目管理文件并继续推进。

| 阶段 | 可提交确认物 | 通过标准 |
|---|---|---|
| A. 项目初始化 | 项目看板、文件夹结构 | 素材分类清楚，最终交付明确 |
| B. 图片注释计划 | 图片双语注释清单 | 中文-only 图片有英文补注计划 |
| C. SFC 时序 | SFC 时序审核稿 | 自动循环顺序和参数映射可用于 HTML |
| D. HTML 初稿 | A4 分页规划、HTML 草稿、手机/PC截图 | A4 分页明确，同一框架下图片占比足够大 |
| E. 反馈吸收 | 修订版 HTML、反馈处理记录 | 已处理反馈，无内部说明外露 |
| F. 最终验证 | 最终 HTML、手机截图、静态审计 | 客户可读，文件路径正确，手机布局通过 |
