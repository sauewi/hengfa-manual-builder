# HTML-Only Delivery

## Fixed Boundary

The customer deliverable is one A4-paged HTML file with relative image assets. Freeze only:

| Field | Decision |
|---|---|
| Final filename | One clear HTML filename |
| Page system | Explicit 210 mm x 297 mm semantic sheets |
| Reading mode | One A4 page system proportionally scaled to the viewport |
| Language | Chinese, English, or bilingual |
| Asset location | Relative project paths |

Do not create a desktop duplicate, PNG, PDF, or another mobile HTML. Conversion belongs outside this skill; preserving one approved source avoids duplicate review and revision.

The same A4 composition serves every device. Do not create a second phone page system or restructure the page at a breakpoint. Improve small-screen readability by enlarging important images within the A4 composition.

## Internal Verification

Screenshots, audit reports, and temporary browser artifacts may be created under `输出/验证` or `临时文件`, but they are not customer deliverables.

Verify:

- the final HTML opens directly;
- every image path is relative and exists;
- requested language content is present;
- the phone viewport is readable;
- A4 composition remains identical on phone and PC;
- detailed primary images occupy at least 75% of A4 content width;
- A4 pages have no clipped or overflowing content;
- no draft or internal wording is visible;
- only one customer-facing HTML is identified as final.
